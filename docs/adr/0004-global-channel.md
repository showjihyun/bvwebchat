# ADR-0004: global 채널 — 예약된 상설 room

- 상태: 승인
- 날짜: 2026-07-17
- 관련 스펙: RQ-04, RQ-12, RQ-13 (입력: answers.md D12·13, 스펙 v1 재리뷰 M-1)

## 맥락

인터뷰에서 global을 "전원 자동 참여 특수 room, room 참여 중에도 항상
수신"으로 결정했다. 재리뷰가 지적한 미해결 모호성: global을 room으로
모델링하면 (a) 전원 접속 종료 시 RQ-12(빈 room 자동 삭제)에 걸리는가?
(b) "global"은 예약 이름인가(RQ-13 이름 고유성과의 충돌)?

## 결정

global은 **서버 기동 시 생성되는 예약된 상설 room**이다:

1. 모든 접속 사용자는 global에 자동 참여하며 탈퇴할 수 없다.
2. **RQ-12 예외**: global은 빈 room 자동 삭제 대상에서 제외 — 참여자가
   0명이어도 존속한다.
3. **RQ-13 예외**: "global"(대소문자 구분 없이)은 예약 이름 — 사용자의
   room 생성 요청에서 거부한다.
4. room에 참여 중인 사용자도 global 메시지를 수신한다(RQ-04).

## 근거

- 특수 room 모델은 데이터 모델·전송 경로(Socket.IO room)를 일반 room과
  통일해 구현을 단순화한다 (인터뷰 D12 결정의 비준).
- 예외 2건의 명문화는 재리뷰 M-1의 해소 — 이 결정 없이 RQ-04/12/13을
  구현하면 각자 추측하게 된다.
- 버린 대안: **별개 개념** — 브로드캐스트 경로·히스토리·참여자 목록을
  이중 구현. **비상설(재생성)** — 삭제/재생성 경합만 늘고 이득 없음.

## 결과

- room 목록(RQ-13)에 global이 항상 표시된다.
- global 히스토리도 ADR-0002 링버퍼(50개) 적용.
- RQ-12/13 테스트는 global 예외 케이스를 포함해야 한다 — 골든 후보:
  "전원 종료 후에도 global 존속", "'GLOBAL' 이름 room 생성 거부".
