# ADR-0001: 실시간 전송 계층 — Node.js + TypeScript + Socket.IO

- 상태: 승인
- 날짜: 2026-07-17
- 관련 스펙: RQ-02, RQ-04, RQ-14, RQ-16 (입력: answers.md B6, E14~16 + 스택 문답)

## 맥락

room 단위 브로드캐스트(RQ-02)와 global 전파(RQ-04), room 내 도착 순서
보장(RQ-14), 동시 100명·단일 서버(RQ-16), 새로고침 재연결(RQ-10)을
지원하는 양방향 실시간 전송이 필요하다. 1인 개발·7/31 마감.

## 결정

백엔드는 **Node.js(LTS) + TypeScript**, 실시간 전송은 **Socket.IO**.
프론트엔드는 **React + Vite + TypeScript**, 클라이언트는 socket.io-client.

## 근거

- Socket.IO의 room 개념이 RQ-01~04와 1:1로 매핑 — room 참여/브로드캐스트를
  직접 구현하지 않는다.
- 자동 재연결 내장 — RQ-10 새로고침 복원과 유예기간(ADR-0003) 구현이 쉬워진다.
- FE/BE 단일 언어(TS) — 1인 개발의 컨텍스트 전환 비용 최소, 메시지 shape
  타입 공유로 경계면 버그 예방.
- 단일 서버 이벤트 루프에서 room 내 도착 순서가 자연 보장(RQ-14).
- 버린 대안: **순수 WebSocket(ws)** — room·재연결·하트비트 직접 구현 비용이
  마감 대비 큼. **SSE + POST** — 양방향 채팅에 우회적. **Python FastAPI** —
  FE와 언어 이원화로 1인 개발 비용 증가.

## 결과

- Socket.IO 프로토콜 종속 — 순수 WS 클라이언트로는 접속 불가 (수용).
- 순서 보장이 단일 프로세스 전제에 기댐 — 멀티 서버 확장 시 어댑터
  (redis-adapter 등)와 순서 재설계가 필요. RQ-16(100명)에서는 불필요.
- 테스트는 ADR-0005의 전송 대역 정책(인메모리 서버 인스턴스)과 결합.
