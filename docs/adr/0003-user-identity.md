# ADR-0003: 사용자 식별 — 닉네임 + 서버 발급 세션 토큰

- 상태: 승인
- 날짜: 2026-07-17
- 관련 스펙: RQ-10, RQ-12, RQ-18 (입력: answers.md A1~A3, 스펙 v1 리뷰 m-2·N-5)

## 맥락

계정 없이 닉네임만으로 식별하되(RQ-10), 중복 닉네임 자동 고유화와
새로고침 후 동일 사용자 인식·room 복원이 필요하다. 리뷰에서 두 가지
미결정이 지적됐다: 클라이언트 보관 식별자의 신뢰 경계(m-2), 새로고침의
일시 단절이 "마지막 참여자 퇴장 → room 자동 삭제"를 오발하는 경합(N-5).

## 결정

1. 최초 접속 시 닉네임 제출 → 서버가 고유화(자동 접미사) 후 **서버 발급
   불투명 세션 토큰**(충분한 엔트로피의 랜덤 값)을 반환한다.
2. 클라이언트는 토큰을 localStorage에 보관하고, 재연결·새로고침 시 토큰을
   제시해 동일 세션(닉네임 + 참여 room)을 복원한다.
3. **신뢰 경계**: 서버는 클라이언트가 자칭하는 닉네임을 식별자로 신뢰하지
   않는다 — 식별은 오직 서버 발급 토큰으로만. 세션 상태(토큰→닉네임·참여 room
   목록·**room별 안 읽음 개수**·**활성 room**)는 서버 인메모리(ADR-0002와 일관).

   안 읽음 개수를 서버가 보관하는 이유: 카운트 증가 판정(메시지 도착 ∧ 비활성
   room)에 필요한 정보가 이미 서버에 있어 진실 공급원이 하나로 유지되고, 서버
   재시작 시 토큰 무효화와 카운트 소실이 함께 일어나 유령 배지가 남지 않는다.
   클라이언트(localStorage) 보관도 새로고침은 견디지만, 재시작 후 사라진
   room·메시지에 대한 카운트가 살아남아 ADR-0002의 휘발 모델과 어긋난다.
   (안 읽음 개수는 본인에게만 보이는 사적 UI 상태이므로 위조는 위협 모델에
   없다 — 이 결정의 신뢰 경계와는 별개 사안이다.)
4. **활성 room** (RQ-18의 "보고 있는 room"의 정의): 클라이언트가 room을 열 때
   서버에 통지하는 값으로, **세션당 하나**다. 서버는 통지된 room이 그 세션의
   참여 room인지 검증하고, 아니면 거부한다(GA-10과 동일 원칙 — 격리는 서버가
   강제한다). 창 포커스·탭 가시성은 추적하지 않는다(design-answers D13-r3).
   세션을 공유하는 다중 탭은 **마지막 통지가 이긴다** — 탭별 카운트는 보장하지
   않는다(localStorage 세션 공유의 수용된 한계).
5. **퇴장 유예기간**: 연결 종료 시 즉시 퇴장 처리하지 않고 **30초 유예**를
   둔다. 유예 내 동일 토큰 재연결 시 참여 상태(활성 room·안 읽음 개수 포함)를
   복원하고, 유예 만료 시 퇴장으로 확정한다. RQ-12의 "마지막 참여자 퇴장"
   판정(빈 room 자동 삭제)도 유예 확정 이후에만 수행한다.
   **유예 만료로 퇴장이 확정되면 그 room의 안 읽음 개수도 함께 버린다**
   (RQ-18의 범위가 "참여 중인 room"이므로).

## 근거

- 토큰 방식은 닉네임 위조로 타인 세션을 탈취하는 경로를 차단(m-2 해소)하면서
  계정/비밀번호 없는 인터뷰 결정(A1)을 유지한다.
- 30초 유예는 새로고침(수 초)을 넉넉히 흡수하면서, 이탈 사용자를 room
  참여자 목록(RQ-15)에 오래 남기지 않는 균형값.
- 활성 room을 클라이언트 통지로 받는 이유: 서버는 사용자가 어느 room을 보는지
  알 수 없다(참여 ≠ 열람 — GA-06이 다중 참여를 확정). 통지 없이는 RQ-18의
  카운트 판정이 불가능하다. 다만 통지를 무검증 수용하면 미참여 room의 카운트를
  조작할 수 있으므로 참여 여부 대조를 결정에 포함했다.
- 버린 대안: **닉네임 자체를 식별자로 신뢰** — 위조 취약. **계정+JWT** —
  인터뷰에서 명시 비요구. **유예 없음** — 새로고침마다 room 삭제·재생성
  왕복(N-5 경합 그대로).

## 결과

- 사칭 방지 보장은 없다(누구나 새 세션에서 비슷한 닉네임 사용 가능) —
  무인증 사내 데모의 수용된 리스크로 문서화.
- 서버 재시작 시 토큰 무효 → 전원 새 세션(ADR-0002 소실과 일관).
- 공유 브라우저는 세션을 공유한다 (localStorage 특성, 수용).
- 유예기간은 fake timer로 테스트한다(ADR-0005). 골든 후보: "새로고침 후
  room 복원", "유예 만료 후 빈 room 삭제".
