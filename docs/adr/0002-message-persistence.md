# ADR-0002: 메시지 영속성 — 인메모리 (영속 계층 없음)

- 상태: 승인
- 날짜: 2026-07-17
- 관련 스펙: RQ-11, RQ-14 (입력: answers.md B4·B5)

## 맥락

입장자에게 최근 50개 히스토리를 보여줘야 하고(RQ-11), 재접속 복구는
히스토리로 대체한다(RQ-14). 인터뷰에서 서버 재시작 시 소실을 허용하고
영속 DB를 비요구로 결정했다.

## 결정

메시지는 **프로세스 인메모리**로만 보관한다 — room당 최근 50개를 유지하는
링버퍼(초과 시 오래된 것부터 폐기). 파일/DB 영속 계층을 두지 않는다.

## 근거

- 인터뷰 결정(B5)의 비준. 사내망 데모(RQ-17)·L&L 목적에 저장 인프라는 과잉.
- 히스토리 요구(50개)가 상한이므로 링버퍼로 메모리 사용이 유계 —
  100명·수십 room 규모에서 무시 가능.
- 버린 대안: **파일 저장** — 동시성 처리 비용 대비 이득 없음.
  **SQLite/DB** — 인터뷰에서 명시 비요구.

## 계층 정리 (스펙 v1 리뷰 M-1 해소)

requirements.md RQ-11의 "인메모리 보관" 문구는 인터뷰 결정의 기록이고,
저장 방식의 규범적 결정과 변경 관할은 **이 ADR**이다. 영속화가 필요해지면
RQ-11 개정 + 본 ADR 대체(새 ADR)를 같은 PR에서 수행한다.

## 결과

- 서버 재시작 시 메시지·room·세션 전체 소실 (수용됨 — RQ-11).
- 50개 초과분은 어떤 경로로도 조회 불가.
- room 삭제(RQ-12) 시 해당 버퍼도 즉시 폐기.
- 후회 지점: 데모 중 서버 크래시 = 대화 전체 증발. 재시작 안정성이 중요해지면
  이 ADR부터 대체할 것.
